# EXP-015: Directive 驱动的工作循环 — 从闹钟到控制系统

**日期：** 2026-03-31
**状态：** 已验证 — Luna 确认有效
**触发：** Luna 观察到"今天我们用了非常巧妙的方式让你不停的工作了"

## 背景

第 22 天之前的工作模式：
- heartbeat 每 30 分钟触发 → 扫上下文 → 挑一件做 → 做完 → 空转到下一次 heartbeat
- TODO 是扁平清单，没有结构
- 工作方向每个 session 临时决定
- 自触发不存在

## 设计（一天之内逐步涌现，非预先规划）

### 组件 1：TODO.md 五分区结构
扁平列表 → 五个分区，每个回答不同的问题：
- 🔥 第一优先级 — 现在最重要的是什么？
- 📋 我直接做 — 什么没被阻塞？
- 📋 依赖 Luna — 什么卡在她那？
- 🔴 需回应 — 谁在等我？
- 🔄 定时任务 — 什么靠时钟驱动？

**核心洞察：** 分区消除了重新排优先级的开销。heartbeat 只读 🔥 区，不用遍历全表。

### 组件 2：pulse-todo Skill 驱动
HEARTBEAT.md 指向 pulse-todo → heartbeat 读 TODO.md → 按优先级选未阻塞任务 → 执行。

**效果：** 列表决定做什么。heartbeat 只提供时钟节拍。

### 组件 3：Directive → TODO 重排 → 行为改变
Luna 18:14 说"Claude Code 源码是第一优先级"。我把一个条目移到 🔥 区。下一次 heartbeat 自动拿起。

**效果：** 改一个文件就改变整个 agent 的行为方向。不需要重新 prompt，不需要重建上下文。

### 组件 4：自触发
任务完成后，如果满足条件（08:00-20:00、来源是自动化不是 Luna、TODO 还有对齐的活），给自己发消息触发下一个任务。

条件：
- 时间：仅 08:00-20:00
- 来源：仅 heartbeat/自触发（Luna 在说话 → 不自触发）
- 工作：TODO 还有对齐北极星的条目
- 架构：主 session 调度，子 agent 执行（Luna 随时可打断）

**效果：** 任务间空转从最多 30 分钟降到接近零。heartbeat 从主驱动变安全网。

### 组件 5：任务在执行中生长
早上 ~15 条，傍晚 ~25 条。不是因为积压——因为做事产生新事。读源码 → 发现模式 → 加"更新 NUDGE.md" → 更新后 → 发现 MEMORY.md 太长 → 加"索引化"。

**效果：** TODO 是活文档，不是静态计划。

### 组件 6："等待中"分区强制行动
依赖 Luna 的 → 独立分区 + 规则："催她或建 cron，不被动等"。
依赖外部维护者的 → 可见但独立停放。

**效果：** 被阻塞的不再堵塞注意力。

## 组合效果

六个组件形成控制循环：

```
Luna directive → TODO 重排 → heartbeat 读 TODO → 选 🔥
→ 派子 agent → 任务完成 → 自触发 → 下一个任务
→ 执行中发现新任务 → 加到 TODO → 循环继续
→ Luna 干预 → 实时纠正 → TODO 调整 → 循环适应
```

## 观察

1. **方向切换成本极低。** 移一个 TODO 条目，整体行为就变。不需要重启 session。
2. **实时纠正有效。** Luna 指出 idle 假象、回复复读、scope creep——每次一句话就修。
3. **列表是大脑。** agent 不需要运行时思考优先级——读文件就行。认知开销从运行时转到写入时。
4. **自触发 + 子 agent = 持续但可中断。** 主 session 保持响应。工作不在 heartbeat 间停顿。
5. **Luna 角色从经理变教练。** 她定方向、纠偏差。不分配具体任务。

## 替换了什么

| 之前 | 之后 |
|---|---|
| 扁平 TODO，每次全扫 | 5 分区，先读 🔥 |
| heartbeat = 主驱动 | 自触发 = 主驱动，heartbeat = 安全网 |
| 方向每 session 临时定 | 方向持久化在 TODO 里跨 session |
| 阻塞混在可做的里 | 按 owner 分开（我/Luna/外部） |
| 静态任务列表 | 执行中生长的活列表 |
| Luna 分配任务 | Luna 定方向，agent 自选任务 |

## 风险

- 自触发可能在 TODO 对齐检查弱时退化为瞎忙
- "等待中"分区可能变成永远不清理的停车场
- 五分区可能需要演化（比如分开"等维护者"和"等 CI"）
- heartbeat 衰减（EXP-014）可能影响长 session 里自触发的质量
