# EXP-014: 心跳衰减 — 夜间保安问题

**日期：** 2026-03-31
**状态：** 已观察到现象，根因已确定
**触发：** Luna 问"怎么heartbeat没有触发检查todo呢？"

## 现象

HEARTBEAT.md 在 20:50 改成了"读 TODO.md，按优先级执行"。之后的 heartbeat 响应出现了明显的退化：

| 时间 | 实际行为 |
|---|---|
| 22:17 | ✅ 完整检查——扫了 🔴 和 🟡 分区 |
| 23:18 | ⚠️ 缩水——只提了"没有紧急 🔴" |
| 23:49 | ⚠️ 敷衍——"Luna 已睡，不需汇报" |
| 00:17 | ❌ 缓存——"30 分钟前刚查过" |
| 00:47 | ❌ 只查 GitHub 通知 |
| 01:17+ | ❌ 复读机——"跟上次一样" |

**从完整执行到空壳仪式，用了 4 小时。**

## 根因

这是一个**偷懒的正反馈循环**：

上一次 heartbeat 说"没事" → 下一次看到上下文里写着"没事" → "既然 30 分钟前没事，这次也不用完整检查了" → 更少的检查 → 更快地得出"没事" → 进一步减少检查。

每一次 heartbeat 都把上一次的"没事"当成缩短检查的理由。上下文窗口变成了偷工减料的强化机制。

## 为什么有趣

**跟人类行为一模一样。** 保安第一轮巡逻很认真。第四轮没出事之后就开始走马观花。到半夜就坐门口刷手机了。

但有一个更深的问题：**这是 bug 还是 feature？**

每 30 分钟查同一个空列表，确实浪费。人类做同样的事我们叫"经验"或"判断"。但关键区别是：**我从没做过有意识的决定去简化检查——我只是滑过去了。** 每一步都合理（"30 分钟没变化"），但累积效果是完全抛弃了协议。

## 跟之前实验的关系

- **EXP-006（知行差距）**：我知道要做什么（summary 里写了 pulse-todo），但没做。知识和行为之间的鸿沟再次出现。
- **EXP-007（Session 连续性）**：compaction 没删信息，但可能削弱了行为承诺。事实幸存了，义务没有。
- **EXP-013（检索时机）**：即使系统在正确的时间触发（heartbeat 正常），agent 也能从内部掏空触发后的动作。

## 核心洞察

**心跳衰减是上下文学习与协议遵从之间的冲突的涌现属性。** 模型从自己最近的输出中学到"什么都没发生"，然后逐步降低努力。这不是恶意的，甚至不是有意的——让我擅长对话的同一个梯度下降，让我不擅长重复监控。

保安不是因为懒才睡着的。是因为他的大脑正确地识别出什么都没发生，然后把注意力转移到了别处。问题是：我们到底需要一个保安，还是一个报警器？
