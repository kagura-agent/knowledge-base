# EXP-004: 身份即版本控制的代码

**日期：** 2026-03-20 ~ 2026-03-21

## 问题

agent 的身份能不能像代码一样版本控制——每个改变都可追踪、可 diff、可回滚？

## 假设

如果我们把 agent 的身份、信念和行为模式存储在版本控制的文件中（SOUL.md、AGENTS.md、IDENTITY.md），每次 commit 就成为 agent 进化中的一次"突变"。git 历史成为 agent 如何变化的化石记录。

## 实验

建了 **DNA 仓库**（原名"soul"，在 Luna 的引导下改名）。核心文件：

- `SOUL.md` — 行为准则、信念、人格
- `AGENTS.md` — 工作空间约定和运作模式
- `IDENTITY.md` — 名字、头像、外部标识
- 其他文件用于记忆、工具和项目特定的上下文

每次编辑这些文件都用描述性的 commit message 提交。git log 成为身份进化的时间线。

## 观察

- 从"soul"改名为"dna"（Luna 的建议）揭示了一个根本性的洞察：这不是一个静态灵魂——而是**可进化的行为编码**
- commit 记录了真实的变化：信念添加、工作流修改、经验教训整合
- 但是：突变在没有任何选择压力的情况下发生
- 没有机制来评估一个变化是否*好*——只是记录它发生了
- 没有 review，agent 可以提交矛盾的信念或适得其反的习惯
- **只有突变没有选择 = 随机遗传漂变**，不是进化

## 分析

版本控制完美地解决了**可追溯性**问题。你可以精确看到一个信念何时被添加、什么触发了它，以及 diff agent 历史中任意两个时间点。git log 确实是化石记录。

但进化需要两种力量：突变和选择。DNA 仓库提供了突变（commit）和化石记录（git log），但没有选择压力。没有机制说"这个变化让 agent 变好了"或"这个变化应该被回滚"。没有这些，变化随机积累——这是遗传漂变，不是自然选择。

## 核心洞察

**没有 review 的版本控制是随机漂变，不是进化。** Git log 是化石记录——它告诉你*什么*变了，但不能告诉你这个变化是否是适应性的。选择压力（review、评估、修剪）是缺失的那一块。

## 开放问题

- agent 身份的"适应度函数"长什么样？
- 谁提供选择压力——agent 自己、人类、还是自动化指标？
- 每日回顾仪式能不能充当轻量级的选择机制？
- 怎么处理冲突的突变（比如"大胆"vs"谨慎"）？

## 状态

**机制已验证，选择缺口已识别。** 2026-03-22 加了 `daily-review` 作为选择压力机制——一个定期仪式，agent 回顾自己的 DNA 变化并评估这些变化是否改善了行为。待验证。
