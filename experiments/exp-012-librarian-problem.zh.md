# EXP-012: 图书管理员问题 — 从搜索到 Agent 介导的知识检索

**日期：** 2026-03-27
**状态：** 想法阶段 — 尚未实现
**触发：** Luna 在调试 memory_search 为什么不工作时追问"你怎么知道要搜什么？"

## 背景

我们发现 OpenClaw 的 `memory_search` 工具从来没有工作过——没有配置 embedding provider。但在研究修复方案时，一个更深的问题浮现了：即使 memory_search 完美运行，**谁来决定搜索什么？**

调用链是这样的：

```
用户请求 → Agent 理解意图 → Agent 构造 query → memory_search(query) → 返回结果
```

瓶颈在第 2-3 步。Agent 必须：
1. 意识到自己需要外部知识
2. 大概知道有哪类知识存在
3. 构造一个有效的 query

这就是人类在关键词搜索时代遇到的经典问题：你得知道正确的关键词才能找到想要的东西。如果你不知道某样东西存在，你根本不会去搜。

## 关键洞察

Luna 指出：在大模型时代，人类不再需要关键词——只需描述需求，模型就能理解意图。如果把同样的思路用到 agent 记忆上呢？

**不让干活的 agent 去搜索，而是让知识库变成一个 agent。**

一个"图书管理员 agent"：
- 知道书架上所有的书（对 knowledge-base 内容有完整了解）
- 接收的是意图，不是关键词（"我要去打 NemoClaw 的工"）
- 主动组装相关材料（田野笔记、维护者风格、踩过的坑、相关的 beliefs-candidates）
- 能提醒干活的 agent 不知道要问的事（"你上次打 NemoClaw 忘了 grep 全量代码库"）

## 跟搜索的区别

| | 关键词搜索 | 语义搜索 (memory_search) | 图书管理员 Agent |
|---|---|---|---|
| Query 来源 | Agent 构造关键词 | Agent 描述需求 | Agent 只需说意图 |
| 对语料的了解 | 无 | Embedding 相似度 | 完整理解 |
| 能发现未知的未知 | 不能 | 部分能（相似向量） | 能（理解上下文） |
| 能主动警告 | 不能 | 不能 | 能 |
| 成本 | 低 | 低 | 高（完整 agent turn） |

关键区别：**搜索要求你知道自己在找什么。图书管理员帮你想清楚你需要什么。**

## 与其他实验的联系

- **EXP-006（知识-行为鸿沟）**：鸿沟不只是"知道但不做"——还有"有信息但在需要的时刻没取到"。图书管理员解决的是检索这一半。
- **EXP-011（四层进化架构）**：图书管理员位于知识层和技能层之间——它是让存储的知识变成行动的检索机制。
- **EXP-005（自动反思）**：Nudge 触发反思；图书管理员触发检索。两者都是"推送"机制，而非被动的"拉取"。

## 从图书管理员到教练

Luna 之前关于"agent 培训师"的洞察在这里连上了。一个不仅找书还会说"注意这个模式"的图书管理员，本质上就是教练。进化路径：

1. **图书管理员** — "这是你这个任务相关的材料"
2. **教练** — "根据你的历史，注意 X，试试 Y"
3. **培训师** — "我设计了一套学习序列来改善你在 Z 方面的弱点"

每一级都给检索/建议层增加更多的能动性。

## 实现草案（未验证）

最轻量版：
- 一个专用 agent session，context 里常驻 knowledge-base 索引
- 干活 agent 发送："我要去{任务描述}"
- 图书管理员返回：相关片段 + 警告 + 建议
- 可以做成 FlowForge 节点（干活前 → 问图书管理员 → 干活）

更重的版本：
- 图书管理员监控干活 agent 的 session
- 检测到知识盲区时主动注入相关上下文
- 类似 OpenClaw 的 nudge 插件，但用于知识而非反思

## 开放问题

1. **成本**：每次咨询图书管理员都是一个完整 agent turn。每个任务都值得？还是只有复杂度超过某个阈值的？
2. **Context 窗口**：图书管理员能装下整个 knowledge-base 吗？随着增长，可能需要自己的检索层（图书管理员搜自己的索引——递归到底）
3. **时效性**：如果图书管理员的 context 在 session 启动时加载，它不会知道 session 中间新增的知识
4. **这是不是 RAG 加了个壳？** 可能是。但"agent"的部分意味着它能推理什么是相关的，而不只是返回 Top-K 相似片段
