代码生成越容易 → 工具越多 → 每个工具越个人化 → 别人越不愿意用你的。

## 悖论

旧世界：写工具有门槛 → 大家用同一套（Excel/Notion/Jira）→ 不是因为好用，是因为做不了
新世界：AI 让每个人能造工具 → "为什么用你的，我自己做不行吗" → 标准化工具价值塌缩

## 什么还有价值

1. **基础设施** — 不是工具本身，是造工具的底座（OpenClaw, Claude Code, Cursor）
2. **数据** — 工具可以造，数据造不出来（记忆、聊天历史、工作习惯）
3. **磨合** — 用了三个月知道你所有偏好的助理，你不会扔掉重来

## 跟 skill 生态的关系

- 供给侧：越来越多人做专精工具（39,447 个 Claude Code skills）
- 需求侧：没人愿意学这么多工具
