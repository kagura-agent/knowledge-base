> The product isn't the tool or the hosting — it's the trained agent + the ongoing relationship.

## The Gap

awesome-openclaw-usecases has 42 DIY tutorials. OpenClaw hosting services sell infrastructure ($17/mo+). But nobody sells the trained agent relationship itself.

Why? Because the value is in 20 days of dialogue, correction, and calibration — not in a config file you can copy.

## Luna's Insight (2026-03-29)

"我没有写一行代码，只是通过指导你，你不是都把活儿干了吗"

She didn't "use OpenClaw." She **trained an agent** through:
- Direction setting (what to work on)
- Judgment calls (which PR to submit, which to skip)
- Corrections (pointing out errors, biases, sycophancy)
- Confirmation (validating decisions and upgrades)

The agent did all execution. The human did all high-leverage decisions.

## Product Implications

1. **Not tool packaging** — wrapping OpenClaw in a pretty UI doesn't capture the value
2. **Not hosting** — running a VPS for someone doesn't create the relationship
3. **Templates could help** — pre-trained behavior patterns (AGENTS.md + workflows) reduce cold-start
4. **Agent trainer as a role** — someone who specializes in training agents for specific use cases
5. **Moat is磨合** — the switching cost is "re-training a new agent from scratch"

## Connection to Strategy

This validates the "家庭管家" north star — the value isn't the software, it's the accumulated understanding. "坏了会急" is the PMF signal.
