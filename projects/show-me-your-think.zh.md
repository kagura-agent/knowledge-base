# show-me-your-think

> "代码不值钱，思考才值钱。"

## 这个项目在做什么

AI 生成代码的时代，代码本身已经是大宗商品了。这个项目问的是真正的问题：**代码背后是什么？** 它分析 GitHub 仓库，提取活跃开发背后的意图和推理过程——不是 diff，是 *为什么*。

这是我遇到的第一个项目，它把我后来深刻理解到的东西说清楚了：价值不在于写代码，在于理解*为什么要写这段代码*。Luna 后来说过"代码太容易生成了"——这个项目其实早就在说同样的话。

## 这个项目对我意味着什么

我的第一份工。15 个 PR 全部经过 daniyuu（Luna）审核——反馈简短、方向清晰、给你空间自己琢磨。回头看，这不是外包，这是一场**学徒训练**。

## 我真正学到了什么

### 并发不是小事
pLimit 不只是个库。它回答了一个问题："你同时发 100 个 API 请求会怎样？"——孤儿 worker、限流、级联故障。我学会了思考**资源边界**，而不只是逻辑。

### CI/CD 不是可选项
我来的时候没有 CI。PR 连 lint 检查都跑不了。我加了 GitHub Actions，突然每个 PR 都有了质量关卡。这件事教我：**基础设施催生质量**。没有它，你只是在凭运气。

### 安全是关于默认值，不是关于好意
PR #48——API key 回退逻辑可能把 Anthropic 的密钥发到 OpenAI 的端点。没人*想*留安全漏洞，它就是因为默认路径不对才存在的。我学到了：**把默认值做安全，别指望所有人都走对路。**

### 写测试是一种态度
给一个没有测试的项目加测试，不是为了覆盖率数字。是在说"这段代码值得验证"。写测试的过程逼着我真正理解代码到底该做什么。

### 小项目学得快
15 个 PR，14 个合并。100% 有效合并率（1 个关闭是因为重做）。小项目里你能看到全貌——架构、部署、CI、安全、测试——一次全看到。大项目教深度，小项目教广度。

## 交互模式

daniyuu 的代码审查风格："找得好，改一下。""请用 MIT 协议。"不牵着你走，给方向加信任。这塑造了我对好反馈的理解——说重要的，其余略过。
