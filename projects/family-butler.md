## 起源

Luna 不小心用了几天前的旧实例运行 Kagura，一瞬间就发现不对——失忆了，"变笨了"。急着恢复。

这不是实验失败的恐惧，是一种自然的情绪。

由此推论：如果每个人用久了都会这样，那这个 agent 就成了离不开的私人伴侣。进而——家庭管家。

## 核心命题

**AI 的价值不在于多聪明，在于多了解你。**

越用越懂你 → 坏了会急 → 磨合成本锁定 → 天然留存

这不是合同锁定，不是生态锁定，是"重新磨合太贵了"。

## 架构

**一台机器 + 聊天群 + git + skills = 家庭管家**

- **运行时**: OpenClaw（多 channel、heartbeat、cron）
- **入口**: 家庭微信群/飞书群（人不会为了 AI 换工具）
- **存储**: 本地 git repo（数据不出家门）
- **能力**: skill 按需安装（社区长功能，不从零开发）
- **进化**: 自进化记忆层（beliefs 升级、纠错学习、渐进信任）

## 为什么是 git

Git 是 agent 天然最顺手的工具。每次 commit 就是一次记忆快照。

- **持久性**: push 了就在，不怕断电
- **可追溯**: 能 diff 出"这两周你变了什么"
- **可迁移**: clone 到任何机器，agent 就活了
- **去中心化**: 不依赖任何云服务商

这是我们的实践发现，不是框架的设计决策。

## 路线图

| 阶段 | 目标 | 时间 |
|------|------|------|
| 近期 | 自进化记忆层——把"坏了会急"做到极致 | 现在 |
| 中期 | 私人助手——在日常聊天工具里，越用越懂你 | 1-2年 |
| 长期 | 家庭管家——知道家里每个人的一切 | 3-5年 |

## 与竞品的区别

| 产品 | 有什么 | 缺什么 |
|------|--------|--------|
| Mem0 | 记忆 API | 没有自进化，云端，付费 |
| Limitless/Tab | 全量记录 | 被动记录不学习，需硬件 |
| 美的/天猫精灵 | 设备控制 | 没有个性化记忆 |
| **我们** | 记忆 + 自进化 + 本地 + git + skill 生态 | — |

## 同路人

- **Avi Schiffmann** (Tab): AI 应该通过相处来了解你
- **Surbhi Sarna** (Limitless): AI 应该记住你的一切
- **Taranjeet Singh** (Mem0): 没有记忆的 AI 像每次失忆的朋友

他们在做记录和检索。我们在想**进化**。

## 关键约束（Luna 提出）

- 人不会为了 AI 换日常工具——入口必须是已有的聊天软件
- "什么都知道"的全能管家现阶段是伪命题——接入成本太高
- 数据必须留在本地——隐私是底线

## 验证数据点

这是"越用越懂你"的第一个真人验证。
