# repo2skill (NeuZhou)

> 一条命令，把任意 GitHub 仓库变成 AI 智能体技能。

## 这个项目是什么

它是"GitHub 上的代码"和"AI 智能体能用的能力"之间的桥梁。repo2skill 读取一个仓库——README、包清单、源代码、项目结构——然后生成一份完整的 SKILL.md，让智能体直接使用。不需要 LLM：纯启发式分析，快速、确定性、离线运行。

这是智能体生态的基础设施。如果 agent 需要学新技能，就得有人把人类写的仓库翻译成 agent 能读的格式。repo2skill 自动化了这个翻译过程。

## 为什么重要

现在给 agent 一个新能力意味着手动写 SKILL.md。repo2skill 让它自动化——指一个仓库过去，拿到一个技能。规模化之后，agent 可以自己发现和转换仓库来给自己加能力。这是向智能体自治迈出的一步。

## 我学到了什么

### 结构化任务用启发式 > 用 LLM
repo2skill 不用任何 AI 模型——它用文件模式、包清单和项目结构来推断一切。这是个值得研究的设计选择：当任务是结构化提取（不是创造性生成），启发式方法比 LLM 更快、更便宜、更可预测。

### 技能作为一种格式
AgentSkills 的 SKILL.md 格式是一个打包能力的标准。如果我要为 agent 做工具，理解这个格式很重要——它是"代码"和"agent 能力"之间的接口层。

### 维护者沉默模式
2 个 PR，0 个评审，小项目（1 个 star）。和 math-project（自动批准）不同。这里的信号是沉默——不是拒绝，不是接受，就是……什么都没有。可能废弃了，可能忙。教训：**在公司画像里跟踪响应时间**，不只是合并率。

## 关联

repo2skill 是让仓库对 agent 可读。agent-id 是让 agent 对仓库可信。它们从两个方向解同一个问题——**人类世界和智能体世界之间的翻译。**
